Базовий набір команд для роботи з гітом.

git init - пишиться один раз при створенні проекту для ініціалізації
git status - відображає змінені файли
git add . - команда додає до коміту файли, крапка вказує на всі файли
git remote add origin *url* - додає репозиторій origin. *url* - посилання
git commit -m "commit name" - Створює новий коміт з доданих файлів під назваю commit name
git push -u origin develop - пушить все на гітхаб.

Робота з гілками

git clone *url* - Клонує репозиторій. *url* - посилання
git branch - Показує список гілок
git branch branch-name - Створює нову гілку на якій ти будеш працювати
git checkout branch-name - переход на гілку
Ім'я гілки слід вибирати відповідно до своєї таски.
Перед початком роботи виконати git pull - щоб оновити зміни з віддаленого репа у себе локально

Створення пул реквесту

Після того як пушиш свій проект потрібно зробити пул реквест.
Учасники проекту перевірять код і або залишать коментарі стосовно того що підправити, або затвердять проект і з'єднають його з головною гілкою.

Приклад того як краще називати коміти.
